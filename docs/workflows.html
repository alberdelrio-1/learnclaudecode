Real-World Workflows & Examples

Common tasks and how to accomplish them with Claude Code.

1. Code Exploration & Understanding

Get up to speed quickly on an unfamiliar codebase.

Scenario

You've joined a new project and need to understand the codebase structure.

Step-by-Step

1

Start with High-Level Overview

Give me an overview of this codebase. What's the main technology stack and how is the code organized?
2

Explore Specific Areas

Show me how user authentication works in this project. Walk me through the flow from login to session management.
3

Find Key Components

Where are the API endpoints defined? Show me examples of how data is fetched from the backend.

Result

You understand: Project architecture, key patterns, where to find specific functionality, and how components connect.

2. Bug Fixing

Diagnose and fix issues efficiently.

Scenario

Users report an error when submitting a form.

Step-by-Step

1

Share Error Information

I'm getting this error when users submit the contact form:
"TypeError: Cannot read property 'email' of undefined"

Here's a screenshot of the error [drag image]

Can you help me diagnose and fix this?
2

Claude Investigates

Claude will:

  • Find the form submission code
  • Identify where the error occurs
  • Understand the data flow
  • Propose a fix
3

Review and Test Fix

That looks good. Now run the tests to make sure nothing else broke.

Pro Tips

  • Always share error messages (don't paraphrase)
  • Include screenshots when helpful
  • Mention steps to reproduce
  • Ask Claude to explain the root cause

3. Feature Development

Build new features from scratch.

Scenario

Add a dark mode toggle to your application.

Step-by-Step (Using Plan Mode)

1

Request in Plan Mode

I want to add a dark mode toggle to the application.
Requirements:
- Toggle button in the header
- Persist user preference
- Smooth transition between themes
- Use existing color scheme conventions

Please create a plan first before implementing.
2

Review Plan

Claude presents approach:

  • Create theme context/state
  • Add CSS variables for colors
  • Build toggle component
  • Implement localStorage persistence
  • Update existing components

Approve or request changes

3

Claude Implements

Claude creates/modifies files as planned

4

Test and Refine

The dark mode works! But can you make the transition smoother? Currently it's jarring when switching.

4. Code Refactoring

Improve code quality while maintaining behavior.

Scenario

Legacy code using outdated patterns needs modernization.

Example Request

The files in /src/utils/ use old callback patterns.
Please refactor them to use async/await instead.

Make sure to:
- Keep all existing functionality
- Update tests accordingly
- Follow our existing code style (check CLAUDE.md)
- Run tests after each file to ensure nothing breaks

Claude Will:

  1. Identify files with callback patterns
  2. Refactor one file at a time
  3. Update corresponding tests
  4. Run test suite after each change
  5. Report any issues found

5. Writing Tests

Generate comprehensive test coverage.

Example Request

Create comprehensive tests for @src/services/auth.ts

Include:
- Happy path scenarios
- Edge cases (expired tokens, invalid credentials)
- Error handling
- Mock external dependencies

Follow the test patterns in our existing tests.

Best Practices

  • Reference existing test files for patterns
  • Be specific about what to test
  • Ask for edge cases explicitly
  • Request test coverage metrics

7. Git & Pull Requests

Manage version control with Claude's help.

Creating a Commit

Create a git commit with all my changes.
Write a clear commit message that explains what was done and why.

Creating a Pull Request

Create a pull request for my dark-mode feature branch.
Include:
- Summary of changes
- Testing checklist
- Screenshots of the new dark mode

Use the gh CLI tool.

Claude will:

  • Analyze all changes in your branch
  • Write descriptive PR summary
  • Create testing checklist
  • Push and create PR via gh CLI
  • Return PR URL for review

1. Analyze the dashboard code for potential issues
2. Check for race conditions or async bugs
3. Look for error handling gaps
4. Suggest debugging strategies

  2. Identify potential race conditions
  3. Check async operation handling
  4. Review error boundaries
  5. Suggest logging additions
  6. Recommend debugging tools

Best Practices for All Workflows

✅ Do This

  • Be specific about requirements
  • Reference files with @filename
  • Provide error messages verbatim
  • Share screenshots when helpful
  • Mention constraints explicitly
  • Ask Claude to explain its approach
  • Use plan mode for complex tasks
  • Interrupt and redirect if needed (Esc)

❌ Avoid This

  • Vague requests ("fix the bugs")
  • Paraphrasing error messages
  • Assuming Claude knows context
  • Long conversations without /compact
  • Mixing unrelated tasks in one session
  • Not reviewing changes before approval
  • Forgetting to check CLAUDE.md exists
