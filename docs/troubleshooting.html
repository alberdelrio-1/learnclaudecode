Troubleshooting & Quick Reference

Solutions to common issues and quick reference guides.

Start Here: /doctor Command

Before troubleshooting manually, run the diagnostic tool:

/doctor

This automatically checks:

  • Installation status
  • Settings configuration
  • MCP server connections
  • Context usage
  • Permission configuration

Common Issues & Solutions

🚫 Installation & Login Issues

Problem: "command not found: claude"

Solution:

  1. Check if installation completed successfully
  2. Restart your terminal
  3. Verify PATH configuration: 
    echo $PATH
  4. Reinstall if necessary

Problem: OAuth/Authentication Errors

Solution:

  1. Verify account access (Pro, Max, Team, or Enterprise required)
  2. Check internet connection
  3. Try logout and re-login: 
    claude logout
claude
  4. Clear browser cache if using web authentication

⚙️ Configuration Issues

Problem: Settings Not Applying

Solution:

  • Check configuration scope priority (managed > CLI args > local > project > user)
  • Verify JSON syntax is correct (use /config to check)
  • Settings auto-reload, but some require session restart

Problem: Hooks Not Firing

Solution:

  1. Verify JSON syntax in hooks configuration
  2. Check matcher patterns are correct
  3. Ensure hook scripts are executable: 
    chmod +x ./scripts/your-hook.sh
  4. Test hook script independently

Problem: MCP Servers Failing to Connect

Solution:

  1. Run /doctor to check MCP status
  2. Verify credentials are correct
  3. Check server URL is accessible
  4. Re-authenticate if needed: /mcp
  5. Review server logs for specific errors

🔥 Performance Issues

Problem: High CPU/Memory Usage

Solution:

  1. Use /compact to compress conversation
  2. Clear context with /clear and restart
  3. Close unused sessions
  4. Switch to lighter model (Haiku) for simple tasks

Problem: Context Thrashing (Autocompact Running Repeatedly)

Solution:

  • Read large files in chunks, not all at once
  • Use /compact with specific focus
  • Move work to subagents to isolate context
  • Start new session for different tasks
  • Use /heapdump for memory analysis (advanced)

Problem: Slow Responses

Solution:

  1. Check context usage: /usage
  2. Switch to faster model: /model → Sonnet or Haiku
  3. Reduce effort level for simple queries
  4. Check internet connection speed

🔍 Search & File Access Issues

Problem: Search Not Working

Solution:

  1. Install ripgrep (required for search): 
    # macOS
brew install ripgrep

# Windows
winget install BurntSushi.ripgrep

# Linux
sudo apt install ripgrep
  2. Alternative: Set environment variable: 
    export USE_BUILTIN_RIPGREP=0

Problem: Claude Can't Read Certain Files

Solution:

  • Check file permissions (Claude needs read access)
  • Verify file path is correct (use absolute paths)
  • Some binary formats may not be supported
  • Check if file is in .gitignore or excluded by settings

💰 API & Cost Issues

Problem: 5xx Server Errors

Solution:

  • Anthropic service issue - retry later
  • Check status page for known outages
  • Wait a few minutes and try again

Problem: 429 Rate Limiting

Solution:

  • Reduce request frequency
  • Spread out parallel work
  • Upgrade to higher tier plan if needed
  • Wait for rate limit to reset

Problem: Unexpected High Costs

Solution:

  1. Check usage: /usage
  2. Identify what's consuming tokens:
    • Long conversations without /compact
    • Reading many large files
    • Parallel subagents (multiplies costs)
  3. Optimize:
    • Use cheaper models for simple tasks
    • Clear context between unrelated work
    • Read files selectively, not wholesale

🖥️ IDE Integration Issues

Problem: VS Code Extension Not Connecting

Solution:

  1. Restart VS Code
  2. Check extension is installed and enabled
  3. Verify CLI is installed: claude --version
  4. Reload window: Cmd/Ctrl+Shift+P → "Reload Window"
  5. Reinstall extension if needed

Problem: JetBrains Plugin Issues

Solution:

  • Check IDE version compatibility
  • Update plugin to latest version
  • Restart IDE
  • Check IDE logs for errors

Getting Help

1. Run Diagnostics First

/doctor

This provides valuable troubleshooting information.

2. Check Official Documentation

Visit code.claude.com/docs for the latest information.

3. Report Issues

/feedback

Report bugs directly to Anthropic. Include:

  • /doctor output
  • Error messages (exact text)
  • Steps to reproduce
  • Platform and version info

4. Community Resources

  • GitHub Issues: Check for known issues
  • Community forums: Search for similar problems
  • Stack Overflow: Tag "claude-code"

5. Safe Mode

If all else fails, start in safe mode to isolate problems:

claude --safe-mode

This disables custom configurations that might be causing issues.

Cost Optimization Tips

1. Use Right Model

Haiku for simple tasks, Sonnet for daily work, Opus only when necessary

2. Manage Context

Use /clear between unrelated tasks. Run /compact during long sessions

3. Read Selectively

Don't load entire large files. Read specific sections or use search

4. Minimize Subagents

Parallel work multiplies costs. Use only when time savings justify expense

5. Leverage Caching

Cached tokens cost ~10% of normal. Keep sessions active to benefit from cache

6. Monitor Usage

Check /usage regularly to understand patterns and optimize
