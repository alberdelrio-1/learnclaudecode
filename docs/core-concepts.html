Section 1: Getting Started

1. The Terminal

What it is: The text-based interface where Claude Code runs.

Why it matters: Unlike browser chatbots, the terminal gives Claude access to your computer files, images, and apps like Google Suite and Airtable.

For non-coders: Think of it like a command center where you type instructions instead of clicking buttons.

Try this:

mkdir ~/playground
cd ~/playground
claude

This creates a practice folder and starts Claude Code.

2. Installation & Pricing

What it is: Setting up Claude Code and choosing your subscription plan.

Plans available:

  • Max ($100-200/month): Unlimited daily use - recommended for work
  • Pro ($20/month): Limited usage - good for occasional use
  • Free tier: Very limited - mainly for testing

ROI: Business owners typically see return on investment within a week!

3. File Access

What it is: Claude reads and edits actual files on your computer directly.

Why it matters: No copy-pasting required! Ask "tighten my proposal.docx" and Claude modifies it in place, preserving formatting.

Analogy: Instead of reading over your shoulder and telling you what to type, Claude can use your keyboard directly (with your permission).

4. Image & PDF Reading

What it is: Drop screenshots, PDFs, diagrams, or photos into Claude.

Why it matters: Claude reads text from images automatically. No manual retyping needed.

Example uses:

  • Screenshot of an error message → Claude explains and fixes it
  • PDF invoice → Claude extracts data to spreadsheet
  • Diagram → Claude explains what it shows

Section 2: Your First Real Tasks

5. Tool Use (Claude ACTS, Not Just Chats)

What it is: Claude doesn't just chat—it performs actions using "tools".

Actions Claude takes:

  • Searches folders
  • Reads files
  • Runs commands
  • Edits documents

You see each "tool call" happen in real time as Claude works.

For non-coders: This is like having an assistant who doesn't just give advice, but actually does the work for you.

6. How to Talk to Claude Code

Best practices:

  • Be specific: Include tone, constraints, and file names
  • Use @mentions: Tag specific files with @filename.xlsx
  • Start vague: Let Claude ask clarifying questions to refine tasks
  • Provide context: Share error messages, screenshots, examples

Good prompt:

"Make @proposal.docx more professional. Focus on the executive summary. Keep it under 2 pages."

Vague prompt:

"Fix the document"

7. CLAUDE.md (Your AI's Instruction Manual)

What it is: A file containing your business rules, brand voice, and preferences.

Why it matters: Claude reads it automatically in every conversation, eliminating repetition.

What to include:

  • Coding standards and style
  • Build and test commands
  • Brand voice and tone
  • Project-specific gotchas
  • Workflows and procedures

Analogy: Like company guidelines that every employee reads on day one.

8. Plan Mode

What it is: Claude shows you its full approach before executing.

How it works:

  1. You ask Claude to do something
  2. Claude creates a plan
  3. You review and approve (or modify)
  4. Claude executes the approved plan

When to use: Critical for important tasks—ensures you stay in control.

Section 3: How Claude's Brain Works

9. Context Window

What it is: Think of Claude's memory as a whiteboard.

How it works: Everything you write and every response fills the whiteboard. When full, older information gets compressed or forgotten.

Why it matters: Long conversations waste space; short focused sessions keep it clean.

Performance Impact

As the context window fills up, Claude's performance can degrade. Keep conversations focused!

10. Tokens & Cost Management

What are tokens? Tokens measure text usage (roughly 1 token = 4 characters).

Check spending: Type /usage to see your consumption.

Save money by:

  • Switching to cheaper models (Haiku) for simple tasks
  • Using /clear between unrelated work
  • Running /compact during long sessions
  • Installing the context-mode plugin

11. Model Selection (Picking the Right Brain)

Three options available:

Opus

Smartest, most expensive

Use for: Complex reasoning, critical decisions

Sonnet

Fast, affordable, balanced

Use for: Daily driver, most tasks

Haiku

Cheap, quick

Use for: Simple tasks, quick answers

Switch models: Use /model command mid-conversation based on task complexity.

Section 4: Managing Conversations

12. /compact

What it does: Compresses your conversation history into a tight summary.

Why use it: Frees memory space while preserving key details.

When to use: During long sessions when context is filling up.

Usage:

/compact

Claude will summarize everything important and clear old details.

13. /clear

What it does: Wipes your current conversation entirely.

When to use: Switching to totally different topics to prevent context confusion.

Analogy: Like erasing the whiteboard completely and starting fresh.

Warning: This cannot be undone! Make sure you're finished with the current conversation.

14. Session Management

What it is: Sessions persist across computer restarts.

How to use:

  • claude --continue - Resume most recent session
  • claude --resume - Browse available sessions
  • claude --resume name - Jump to named session

Why it matters: Start fresh sessions for new projects to avoid context mixing.

Section 5: Controlling Claude

15. Permission Modes & Settings

What it is: Control how much freedom Claude has to act.

Permission levels:

  • "Ask before every edit" - Safe, slower (good for beginners)
  • "Auto-approve safe actions" - Balanced
  • "Do whatever you need" - Fast, requires trust

Set once: Every new session respects your preferences.

16. Effort Levels

What it is: Tell Claude how hard to think.

Examples:

  • Low effort: "What time is it in Denver?" - Quick facts
  • Normal effort: Most tasks
  • High effort: Complex strategy, critical decisions
  • "ultrathink": Maximum reasoning for one response

Analogy: Like asking your colleague for a quick answer vs. asking for a detailed analysis.

17. Interrupt & Redirect

What it is: Stop Claude mid-task and change direction.

How to do it: Press Escape anytime.

Why it matters: No work is lost—type a new instruction and pivot instantly.

Pro tip: You don't need perfect prompts! Course-correct as Claude works.

Section 6: Reviewing Work & Teaching Claude

18. Visual Studio Code

What it is: Free app for reviewing Claude's work side-by-side.

Shows: Color-coded changes (additions in green, deletions in red).

Why use it: Better than squinting at terminal output. See exactly which lines Claude added, removed, or modified.

19. Memory

What it is: Claude's personal notebook that it writes itself.

Opposite of CLAUDE.md:

  • CLAUDE.md - You write instructions manually
  • Memory - Claude captures preferences automatically

Example: You correct Claude: "I prefer casual tone." Claude saves this and remembers going forward.

View/edit: Use /memory command.

20. Project Scope vs Global Scope

What it is: Different rules for different projects vs. consistent preferences everywhere.

Analogy:

  • Project scope - Department handbooks (specific to one project)
  • Global scope - Company-wide policy (applies everywhere)

Example: Global preference for casual tone, but formal tone for legal project.

Section 7: Skills & Automation

21. Slash Commands

What they are: Quick actions accessed with / prefix.

Essential commands:

  • /help - Show available commands
  • /clear - Clear context
  • /compact - Compress conversation
  • /model - Switch models
  • /usage - Check token usage
  • /memory - View/edit memory

Hidden gems:

  • /insights - Shows usage patterns
  • /btw - Add side notes without interrupting

22. Skills (Your Custom Workflows)

What they are: Saved instruction sets that automate multi-step workflows.

How they work: One command triggers entire processes.

Example: /crosspost

  1. Transcribe video
  2. Write captions
  3. Publish to 7 platforms
  4. Update databases

All happens automatically with one command!

Create skills: Save instructions as .md files in .claude/skills/

23. Hooks (Automated Guardrails)

What they are: Scripts running automatically before/after actions.

Example use case: Quality gate hook that intercepts every post, checking:

  • Character limits
  • Banned words
  • Required tags

Result: Flags issues before publishing, preventing mistakes.

For non-coders: Like autocorrect, but customized for your specific needs.

24. Web Browsing

What it is: Paste any URL and Claude fetches, reads, and analyzes webpage content.

Doesn't just summarize: Claude can create files, compare competitors, extract data.

Example uses:

  • "Compare pricing from these 3 competitors and create a spreadsheet"
  • "Read this documentation and write a summary for our team"
  • "Extract all email addresses from this page"

Section 8: Connecting Claude to the Real World

25. MCP Servers (From Consultant to Employee)

What MCP is: Model Context Protocol bridges Claude to real apps.

Connects to: Google Drive, Slack, Airtable, Stripe, Salesforce, etc.

The difference:

  • Without MCP: Claude says "You should update your spreadsheet"
  • With MCP: Claude actually updates it

Analogy: MCP turns Claude from a consultant (gives advice) into an employee (does the work).

26. Perplexity MCP (AI-Powered Web Research)

What it is: Advanced web research connecting to Perplexity.

How it works: Instead of reading one link you provide, Claude:

  1. Researches broadly across sources
  2. Finds verified information
  3. Cites every fact

Use case: "Research our top 5 competitors and summarize their pricing strategies with sources."

Section 9: Agents, Remote Control & Scheduling

27. Subagents (Automatic Parallel Workers)

What they are: For multi-part tasks, Claude automatically spawns mini-Claudes working simultaneously.

Example: Researching 5 competitors takes 2 minutes instead of 10 with automatic parallelization.

Why it matters: Massive time savings on research and analysis tasks.

For non-coders: Like delegating parts of a project to multiple assistants who work at the same time.

28. Remote Control (Use Claude From Your Phone)

What it is: Start a session on your computer, control it from the Claude mobile app.

How to use:

  1. Start session on computer
  2. Scan QR code with mobile app
  3. Control from anywhere

Analogy: Your phone is a remote; the computer does the work.

29. Scheduled Tasks (/loop)

What it is: Schedule recurring tasks using /loop command.

Examples:

/loop 1h "Check email for urgent messages"
/loop 1d "Check Google Drive for new invoices"

Safety: Tasks auto-expire after 3 days to prevent runaway automation.

30. Version Control with Git

What it is: Every Claude change gets saved as reversible versions.

Benefits:

  • Roll back mistakes instantly
  • See history of all changes
  • Optional GitHub integration for cloud backup
  • Team review of changes

For non-coders: Like "track changes" in Word, but for your entire project.
