Essential Features You'll Use Every Day

Master these key features to become productive with Claude Code.

CLAUDE.md - Your Instruction Manual for Claude

The most powerful feature for consistency: a file Claude reads automatically in every conversation.

What is CLAUDE.md?

Think of it as a permanent instruction manual for your AI assistant. Write your preferences once, and Claude follows them in every session.

What to Include

✅ Do Include

  • Build and test commands
  • Code style preferences
  • Brand voice and tone
  • Project-specific rules
  • Common workflows
  • Things Claude can't figure out from code

❌ Don't Include

  • Things obvious from reading code
  • Generic programming principles
  • Basic language syntax
  • More than 200 lines of text

Example CLAUDE.md

# Project Standards

## Build & Test
- Build: `npm run build`
- Test: `npm test -- --coverage`
- Format: `npm run format`

## Code Style
- Use 2-space indentation
- JSDoc comments on exported functions
- No console.log in production code

## Architecture
- Services in `/src/services/`
- React components in `/src/components/`
- Styles in module.css files

## Testing
- Unit tests in `__tests__/` directories
- Minimum 80% coverage for src/
- E2E tests use Playwright

## Gotchas
- Database migrations must be idempotent
- Never commit .env files
- Build requires Node 16+

Where to Place CLAUDE.md

Priority order (first match wins):

  1. Organization-wide - Managed company policies
  2. Project-level (./CLAUDE.md) - Team-shared, version controlled
  3. User-level (~/.claude/CLAUDE.md) - Personal preferences
  4. Local (./CLAUDE.local.md) - Private notes (add to .gitignore)

Pro Tips

  • Keep under 200 lines
  • Be specific and verifiable
  • Add when Claude repeats mistakes
  • Update after code reviews find issues

Memory System - What Claude Remembers

Two complementary approaches for cross-session retention:

CLAUDE.md

  • You write manually
  • Explicit instructions
  • Version controlled
  • Team-shared
  • Persist forever

Auto Memory

  • Claude writes automatically
  • Learned preferences
  • Stored locally
  • Personal to you
  • Editable anytime

What Auto Memory Captures

  • Build commands and test procedures
  • Debugging patterns you've used
  • Architecture decisions you've made
  • Code style preferences you've mentioned
  • Corrections you've given Claude

Managing Memory

/memory

This command lets you view and edit what Claude remembers about your project.

Storage: First 200 lines of MEMORY.md (or 25KB) loads at session start

Location: ~/.claude/projects/<project>/memory/

Separate: Each project has its own memory

Plan Mode - Review Before Executing

Claude shows you its complete strategy before making any changes.

How It Works

1

Request a Task

Ask Claude to do something complex or important

2

Research Phase

Claude explores code, reads files, understands context

3

Plan Presentation

Claude shows complete approach with all steps

4

Your Review

Approve, modify, or reject the plan

5

Execution

Claude implements the approved plan

When to Use Plan Mode

  • Multi-file changes
  • Unfamiliar codebase
  • Critical features
  • Architectural decisions
  • When you want to learn the approach

Benefits

Prevents solving wrong problem - Research first, then implement

Saves tokens - Avoid iterations from misunderstandings

Teaches you - See how Claude thinks through problems

Context Management - Keep Claude Sharp

Context is your most valuable resource. Manage it well!

Understanding the Context Window

Think of Claude's context as a whiteboard that fills up during conversation:

1. System Instructions

Claude's base knowledge + tool access

2. Project Context

CLAUDE.md + Memory files

3. Conversation History

Your messages + Claude's responses

← This fills up quickly!

Context Management Commands

/compact

Compress conversation, keep key details

Use during long sessions

/clear

Wipe everything, start fresh

Use between unrelated tasks

/usage

Check token consumption

Monitor costs and context

Best Practices

  • Monitor usage regularly - Type /usage often
  • Clear between tasks - Unrelated work in separate sessions
  • Read files in chunks - Avoid loading entire large files
  • Use plan mode - Research separately from implementation
  • Compact proactively - Before context fills completely

Performance Degrades When Full

As context fills, Claude's responses become slower and less accurate. Keep it clean!

Sessions - Organize Your Work

Saved conversations tied to project directories.

Session Commands

claude --continue                 # Resume most recent
claude --resume                   # Browse all sessions
claude --resume session-name      # Jump to specific session
/rename                          # Rename current session

Session Features

  • Auto-save - Transcripts save automatically
  • Branching - Create experimental copies
  • Naming - Descriptive names for organization
  • Retention - 30-day default persistence
  • Resume anywhere - Continue across restarts

Organization Strategy

Separate sessions for:

  • Different features or bugs
  • Research vs. implementation
  • Different projects
  • Experiments vs. production work

Permissions - Stay in Control

Three-tier permission architecture balancing capability with safety.

Permission Levels

Read-Only

No approval needed

  • Reading files
  • Searching code
  • Viewing directories

Bash Commands

Approval on first use

  • Running builds
  • Installing dependencies
  • Git operations

File Modifications

Approval per session

  • Editing code
  • Creating files
  • Deleting files

Permission Modes

  • default - Prompts on first use (balanced)
  • ask - Always ask before action (safest)
  • auto - Background safety checks (fastest)
  • bypassPermissions - Skip prompts except critical ops

Permission Rules

Create custom rules for your workflow:

Bash(npm run build)           # Allow specific command
Bash(npm *)                   # Allow with wildcards
Read(./.env)                  # Deny reading secrets
WebFetch(domain:example.com)  # Domain filtering

Rule Evaluation: First match wins!

  1. Deny rules (blocking)
  2. Ask rules (confirmation)
  3. Allow rules (unrestricted)
