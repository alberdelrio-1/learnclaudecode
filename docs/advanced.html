Unlock Claude Code's full potential with these advanced capabilities.

Note: These features are powerful but require understanding of the basics. Make sure you're comfortable with core concepts and essential features first.

Custom Skills - Automate Your Workflows

Skills are reusable commands that automate multi-step workflows with a single slash command.

What Are Skills?

Skills are markdown files containing instructions that Claude executes when you invoke them.

Example: A "/crosspost" skill that:

  1. Transcribes a video
  2. Writes social media captions
  3. Publishes to 7 platforms
  4. Updates tracking database

All triggered with one command: /crosspost

Creating Your First Skill

1

Create Skills Directory

mkdir -p .claude/skills
2

Create Skill File

Create .claude/skills/code-review.md:

# Code Review Skill

Perform a comprehensive code review on the current changes.

Check for:
- Code quality and readability
- Potential bugs or edge cases
- Security vulnerabilities
- Performance issues
- Test coverage gaps
- Documentation needs

Provide specific, actionable feedback with file:line references.
3

Use Your Skill

/code-review

Claude will execute the instructions in your skill file!

Skill Best Practices

  • Be specific about steps to execute
  • Include quality checks and validation
  • Mention output format expectations
  • Reference project standards (CLAUDE.md)

Hooks - Automated Guardrails

Hooks are scripts that run automatically before or after specific events, enforcing policies and automating workflows.

How Hooks Work

Hooks intercept events at specific lifecycle points:

  • Session-level: Execute once per session (start/end)
  • Turn-level: Execute once per turn (prompt submit/complete)
  • Tool-level: Execute on every tool call

Common Use Cases

Policy Enforcement

Block destructive commands like rm -rf

Prevent committing secrets

Permission Automation

Auto-approve safe operations

Require manual approval for critical changes

Context Injection

Load branch names automatically

Include deployment status

Quality Gates

Check character limits

Validate required fields

Flag banned words

Hook Configuration Example

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "./scripts/validate-bash.sh"
          }
        ]
      }
    ]
  }
}

Hook Exit Codes

  • 0: Success - Continue
  • 2: Block the action
  • 1+: Non-blocking error

Subagents - Parallel Work

For multi-part tasks, Claude automatically spawns mini-Claudes working simultaneously.

When Subagents Activate

Claude automatically uses subagents for tasks like:

  • Researching multiple competitors
  • Analyzing several files independently
  • Running parallel tests
  • Delegating side investigations

Example Use Case

Research our top 5 competitors and create a comparison table of their:
- Pricing models
- Key features
- Target markets
- Customer reviews
- Technical architecture

Result:

Instead of researching sequentially (50 minutes), Claude spawns 5 subagents working in parallel (10 minutes total).

Token Cost Consideration

Parallel work multiplies token usage. 5 subagents = 5x the tokens. Use wisely!

Scheduled Tasks & Automation

Run Claude on recurring schedules for autonomous work.

Using /loop Command

/loop 1h "Check email for urgent messages and notify me of any"
/loop 1d "Check Google Drive for new invoices and add to spreadsheet"
/loop 30m "Monitor build status and alert if failures occur"

Scheduling Options

  • 1h - Every hour
  • 1d - Every day
  • 30m - Every 30 minutes
  • 1w - Every week

Safety Feature

Tasks auto-expire after 3 days to prevent runaway automation. Renew manually if needed.

Other Scheduling Methods

  • Routines (Web interface) - Recurring tasks in cloud
  • Desktop Tasks - Scheduled in Desktop app
  • GitHub Actions - CI/CD automation integration
  • Cron Jobs - Traditional Unix scheduling

Advanced Git Integration

Claude Code works seamlessly with git for version control and collaboration.

What Claude Can Do

Basic Operations

  • Create commits with contextual messages
  • Create and switch branches
  • View diffs and change history
  • Stash and apply changes

Advanced Operations

  • Create pull requests with descriptions
  • Resolve merge conflicts
  • Cherry-pick commits
  • Interactive rebasing

Example: Resolving Merge Conflicts

I have merge conflicts after pulling from main.
Can you:
1. Show me the conflicts
2. Resolve them intelligently (keep both changes where possible)
3. Run tests to ensure nothing broke
4. Complete the merge

Git Best Practices with Claude

  • Review commit messages before accepting
  • Ask Claude to explain conflict resolution strategy
  • Use branches for experiments
  • Let Claude write PR descriptions based on actual changes

Batch Processing & /batch Command

Split work across 5-30 isolated subagents, each opening its own PR.

When to Use /batch

  • Updating dependencies across multiple packages
  • Applying refactoring to many files
  • Migrating API versions project-wide
  • Bulk documentation updates

How It Works

/batch "Update all React components to use the new hooks API"

Claude will:

  1. Analyze the codebase
  2. Identify all files needing updates
  3. Split work across multiple subagents
  4. Each subagent creates its own branch
  5. Each opens a separate PR
  6. You review and merge incrementally
