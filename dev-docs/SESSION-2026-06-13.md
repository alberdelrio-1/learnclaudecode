## 6. Agent Teams Feature 🔬

### What Are Agent Teams?

**Definition:** Experimental feature enabling multiple Claude Code instances to work together on complex tasks.

**Key Differences from Subagents:**
- Multiple independent sessions communicate directly
- One session acts as "team lead"
- Teammates maintain separate context windows
- Shared task list with dependency management
- Direct messaging between teammates

### How to Enable

**Method 1: Environment Variable**
```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

**Method 2: Settings Configuration**

Edit `~/.claude/settings.json`:
```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

**Requirements:**
- Claude Code v2.1.32 or later

### Configuration Options

**Display Mode (`teammateMode`):**
- `"auto"` (default): Split panes in tmux/iTerm2, fallback to in-process
- `"in-process"`: All teammates in main terminal, Shift+Down to cycle
- `"tmux"`: Forces split-pane mode

**Model Selection:**
Set in `/config` to specify teammate models

### Usage

**Starting a Team:**
Describe task and team structure naturally:
```
"Create a team of 3 agents to refactor this codebase -
one for tests, one for implementation, one for documentation"
```

**Control Commands:**
- Shift+Down: Switch between teammates (in-process mode)
- Click panes: Direct messaging (split mode)
- Name teammate: Shut down specific teammate
- "End all teammates": Clean up team

### Best Practices

1. **Team Size:** 3-5 teammates optimal
2. **Task Distribution:** 5-6 self-contained tasks per teammate
3. **File Management:** Assign different files to avoid conflicts
4. **Monitoring:** Actively steer underperforming approaches
5. **Permissions:** Pre-approve to reduce interruptions

### When to Use

**Use Agent Teams:**
- Many parallel sessions with centralized monitoring
- Sustained coordination between workers
- Exceeding context window with subagent results
- Ongoing communication needed

**Use Subagents:**
- Verbose output you won't reference again
- Enforce specific tool restrictions
- Self-contained work
- Quick focused delegation

### Known Limitations

- No `/resume` or `/rewind` for in-process teammates
- Task completion status can lag
- Only one team per session
- Teammates cannot spawn their own teams
- Split-pane requires tmux or iTerm2

## 12. Questions Answered

**Q: Can .mcp.json and package.json be moved out of root?**
A: No. npm requires package.json in root, Claude Code requires .mcp.json in root. Solution: Use /docs folder for website (GitHub Pages standard).

**Q: Can Claude Code create NotebookLM notebooks?**
A: Yes! Using the NotebookLM CLI, can create notebooks, add sources, ask questions, and generate content.

**Q: Does NotebookLM have a CLI version?**
A: Yes! notebooklm-py is the full-featured CLI with Python 3.10+, supports all NotebookLM features including content generation.

**Q: How to enable agent teams?**
A: Set environment variable `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` in shell or settings.json. Requires Claude Code v2.1.32+.
